#Hexo是什么？
这段时间一真想弄一个独立的博客系统，但是其他博客的笨重，cnBlog、CSDN又不想用。但发现Hexo时，一下就喜欢上了她，没有任何理由，就依然决定要用她加GitHub来搭建我的博客。

Hexo使用Node编写，支持Markdown, 可生成静态文件，薄板配置灵活。

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。——Hexo官网

安装

因为Hexo运行在Node环境上，所以先安装

  • Node.js
  • Git

接下来使用npm安装hexo

$ npm install -g hexo-cli

什么？你不知道Node.js和Git，那你好好拓展下知识面吧。

快速完成搭建

执行下列命令，Hexo会新建需要的文件。

$ hexo init <folder>
$ cd <folder>
$ npm install

来说说一些重要的文件和文件夹

  • _config.yml 网站的配置信息，在这里配置大部分的参数。
  • package.json 应用程序的信息
  • scaffolds 模版文件夹，当新建文件时，Hexo会根据scaffolds来建立文件。
  • source 资源文件夹，就是你写的文章和html文件，发布时，除posts文件夹隐藏，其他开发的忽略，剩余的会被解析到public文件夹
    • _posts 普通文章
    • pages 独立的页
  • themes 主题文件夹，Hexo 会根据主题来生成静态页面。

此时Hexo已经搭建好了，跑起来看一下。

$ Hexo Server

配置

_config.yml

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
author 您的名字
language 网站使用的语言
timezone 网站时区。Hexo 默认使用您电脑的时区。时区列表。比如说：America/New_York, Japan, 和 UTC 。

常用命令

新建一篇文章 

$ hexo new [layout] <title>

生成静态文件

$ hexo generate
选项 描述
-d, –deploy 文件生成后立即部署网站
-w, –watch 监视文件变动

发表草稿

$ hexo publish [layout] <filename>

启动服务器

$ hexo server
选项 描述
-p, –port 重设端口
-s, –static 只使用静态文件
-l, –log 启动日记记录，使用覆盖记录格式

部署网站

$ hexo deploy
参数 描述
-g, –generate 部署之前预先生成静态文件

渲染文件
|参数| 描述
|-|-|
|-o, –output| 设置输出路径

其他内容请参考官网
https://hexo.io

#发布到GitHub

建立Repository

建立与你用户名对应的仓库，仓库名必须为【your_user_name.github.io】，固定写法。配置公钥（不清楚了去找Git的教程）。

配置和本地文联：
在 _config.yml中 改成我这样子的：

deply:
    type: git

安装hexo-deployer-git

$ npm install hexo-deployer-git --save

修改配置。

deploy:
  type: git
  repo: <repository url>
  branch: [branch]
  message: [message]
